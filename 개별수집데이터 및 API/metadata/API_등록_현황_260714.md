## 4. 공공데이터포털 키 구조

| 키 | 적용 API |
|---|---|
| `DATA_GO_KR_API_KEY` (hex) | 실거래가, 건축HUB 건축물대장, 사업자등록 |
| `OFFICIAL_PRICE_APT_API_KEY` (UUID) | 공동주택가격, 디지털트윈국토/VWorld |
| `OFFICIAL_PRICE_HOUSE_API_KEY` (UUID) | 개별주택가격, 디지털트윈국토/VWorld |
| `OFFICIAL_PRICE_LAND_API_KEY` (UUID) | 개별공시지가, 디지털트윈국토/VWorld |

공시가격 3종은 **데이터셋마다 별도 인증키**이므로 혼용하지 않는다. 또한 여기서 말하는
`서비스 URL`은 API 호출 주소가 아니라 인증키 사용을 허용할 웹 서비스 주소다. 현재 등록값은
`https://www.khug.or.kr/index.jsp`이며, 실제 수집에는 각 가격 데이터의 공식 API URL/레이어명과
key/domain 또는 Referer 처리 방식 확인이 추가로 필요하다.

## 5. API 호출 순서 (위험진단 오케스트레이션)

```
① 도로명주소 정규화 (admCd, 법정동코드)
    ↓
② CODEF 등기부 (권리부담) — 실패 시 mock_registry_*.json
    ↓
③ 건축물대장 (용도, 사용승인일)
    ↓
④ 실거래가 (지역 시세)
    ↓
⑤ 공시가격 3종 중 주택유형에 맞는 API
    ↓
⑥ 사업자등록 상태 + OpenDART (법인 임대인)
```

각 호출 성공 시 응답 JSON을 `raw/{출처}/raw_{출처}_{YYYYMMDD}_{case_id}.json` 형식으로 저장한다.

## 6. 연결 테스트

```bash
cd DIVE2026
python scripts/collect_raw_data.py
```

`scripts/test_api_connections.py`(단순 연결 확인)를 대체하는 `scripts/collect_raw_data.py`가
2026-07-14 추가됐다. 연결 확인뿐 아니라 성공 시 `raw/{출처}/raw_{출처}_{YYYYMMDD}_live.json`,
실패 시 가이드 5장 Fallback 정책대로 `raw/{출처}/raw_{출처}_{YYYYMMDD}_mock_fallback.json`을
자동 저장한다. 결과는 `metadata/api_connection_test_260714.json`에 저장된다.

### 6.1 2026-07-14 실행 결과

2026-07-15 기준 TLS/CA 문제를 수정하고 endpoint를 재정리한 뒤 재실행한 결과, 9개 중 6개가
실 호출에 성공했다. 성공 항목은 도로명주소, CODEF OAuth, 건축HUB 건축물대장, 아파트 매매 실거래가,
사업자등록, OpenDART다. 공시가격 3종은 디지털트윈국토/VWorld 키로 확인되어, 실제 API URL/레이어명
확정 전까지 잘못된 NSDI endpoint 호출을 중단하고 Mock Fallback을 유지한다.
