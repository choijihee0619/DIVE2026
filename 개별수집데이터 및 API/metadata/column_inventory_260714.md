# 발제사 제공 데이터 컬럼 인벤토리

작성일: 2026-07-14

원본 위치: `dive 데이터/` (원본 보존, 수정 금지)

raw 이관 위치: `개별수집데이터 및 API/raw/{hug,auction,dividend,in}/`

## 요약 및 확인사항

- 총 8개 파일, 약 31.6만 행. HUG 사고/대위변제 5개(`raw/hug/`), 경매 1개(`raw/auction/`), 배당 1개(`raw/dividend/`), 상담데이터 1개(`raw/in/`).
- 가이드(`데이터수집_및_API가이드_260714.md` 2.1절)가 예상한 컬럼 중 실제 제공 데이터에 없는 항목: `사고ID`, `사건번호`, `법원`, `감정가`, `최저가`, `낙찰가`, `배당요구종기`, `채권자`, `배당순위`, `임대인유형`. 즉 제공된 경매/배당 데이터는 가이드가 가정한 것보다 컬럼이 단순화되어 있어(상품명·금액·날짜 위주), 케이스 단위로 조인할 고유 식별자(`사고ID`, `사건번호`)가 없다. 파일 간 레코드를 연결할 방법이 없으므로, 전처리 단계에서 각 파일에 자체 UUID(`source_row_id`)를 새로 부여하고 파일 간 직접 조인은 시도하지 않는 것을 권장한다(확인 필요: 발제사에 원문 데이터에 사고ID가 존재하는지 문의).
- `임대보증 사고현황`의 `임대보증금 금액 대비 사고금액 (%)`는 고유값이 1개(100.0)뿐이라 사실상 상수 컬럼이다 — Feature로 사용 시 제외 검토.
- 주소 컬럼(`사업장지역상세`, `시도구`, `물건 소재지`, `시도명`)은 결측률이 0~24%로 파일마다 다르며, 도로명주소 API로 법정동코드 정규화가 필요하다.
- 상담데이터(`raw_in_...xlsx`)는 파일명이 "비식별"이나, 자유서술형 컬럼(`상황요약`, `담당자의견`, `특이사항`)은 결측률 0.3~40.6%이며 자동 스캔(전화번호/주민등록번호 정규식)에서는 검출되지 않았으나 계약일·특정 지역·정황 등 준식별자가 포함되어 있다. RAG 임베딩 전 가이드 11·13장 기준 PII 재검토를 권장한다.
