### 1.2 HUG 안심전세포털 (khug.or.kr/jeonse)

전체 사이트맵 확보. 솔루션에 직접 쓸 자원:

- **RAG 코퍼스 소스(품질 높음, 크롤링 가능 확인)**:
  - 전세사기란 / 전세사기 유형별 사례 및 대처방안(깡통전세·이중계약·신탁사기 등 유형별 스토리+체크리스트)
  - 계약 체결 전/중/후/종료·갱신 시 유의사항 4개 페이지(단계별 체크리스트)

- 피해자 지원 절차(피해접수→피해확인서→경공매 지원)는 **사고 후 사용자 플로우** 설계의 근거.

### 1.3 HUG 빅데이터 개방 포털 HOUSTA (khug.or.kr/index.jsp?mainType=housta)

'전세 및 임대보증' 카테고리 20개 데이터셋 전수 확인. **합성데이터의 한계를 보완할 실제 집계 데이터**:

| 데이터셋 | 활용 |
|---|---|
| 전세보증금반환보증 **발급현황/상세현황** (파일) | **정상 모수(분모)** — 합성데이터에 없는 "전체 발급" 규모 |
| **지역별** 전세보증금반환보증 **사고현황** (파일) | 사고(분자) → **지역별 실제 사고율 산출 가능** |
| 연도별 전세보증금반환보증 서울 사고·대위변제 현황 | 시계열 검증·시각화 |
| **경공매지원서비스 신청자의 전세사기피해주택 소재지(시군구)** | **실제 피해 주택 위치 분포** → 위험지도 레이어 |
| 전세금안심대출보증 신청건수·사고건수·금액 | 상품별 사고율 |
| 임대보증금보증(법인/개인) 발급현황, OPEN API 임대보증금보증 세대수·금액현황 | 임대보증 모수 |
| 주택보증통계 연보 2018~2025 (국가승인통계) | 발표자료 근거 수치 |

- OPEN API 키는 공공데이터포털(data.go.kr) 경유 발급 — 이미 보유한 `DATA_GO_KR_API_KEY` 체계로 신청 가능.
- **핵심 발견**: README 데이터 사전의 최대 한계였던 "정상 계약 대조군 없음 → 사고확률 산출 불가"를, **발급현황(모수) × 지역별 사고현황(분자)** 결합으로 우회할 수 있다. 개별 계약의 사고확률은 여전히 불가하지만, **"지역×상품×기간 실제 사고율"이라는 통계적 사전확률(prior)**을 만들 수 있고, 이것은 합성이 아닌 실데이터라 발표 신뢰도가 높다.

## 3. 통합 컨셉: "전세 생애주기 안심 플랫폼" (가칭: 안심루프)

> **계약 전 예방(소주제②) → 계약 체결 → 거주 중 모니터링 → 사고 후 회수(소주제①)** 를 하나의 루프로 잇고, 4개 역할이 같은 데이터 위에서 각자의 화면을 보는 플랫폼.
> 발제 슬로건 "사고를 막는 데이터, 함께 만들어 주십시오"에 대한 답: **"막고(예방), 남기고(계약 기록), 되찾는(회수) 데이터 루프"**.

### 축 A. 계약 전 예방 — 임차인 (소주제②)

**A1. 주소 입력 → 안심 리포트** (기존 /risk/diagnose 확장)
- 입력: 주소 + 보증금(+ 임대인 유형)
- 산출(점수화·신호등):
  1. 전세가율 신호 — 실거래가·공시가 API 기반(사고사례 분포 84%가 비율 80%↑라는 합성데이터 패턴을 '고위험 패턴 참조선'으로 시각화)
  2. **지역 사고율 신호 — HOUSTA 발급×사고 실집계(신규)**
  3. 주택유형 신호 — 다세대·다가구 사고 집중 패턴
  4. **임대인 신호 — 악성임대인 공개명단 대조(신규), 사업자등록 상태, OpenDART(법인)**
  5. 권리관계 신호 — CODEF 등기부(근저당·압류)
- 하단에 "이 리포트는 사고확률이 아니라 고위험 패턴 충족도" 명시 → 데이터 윤리 어필(심사 가점).
- 결과 화면에서 바로: 계약단계별 체크리스트(포털 콘텐츠) + 챗봇 연결 + 보증가입 안내.

**A2. AI 전세상담 챗봇 + 유사사례**
- 코퍼스: 비식별 상담 938건(사례) + 안심전세포털 전 콘텐츠(공식 절차·제도) + 주택임대차보호법·전세사기특별법(국가법령정보센터).
- 답변 구조: 결론 → 근거 출처 카드(공식자료/유사 상담사례 구분 표시) → 다음 행동 제안(체크리스트, 컨설팅 예약, HUG 이행청구 등) → **근거 없으면 답변 거부 + 상담사 이관 버튼**.
- 분쟁유형·진행단계 자동 분류(텍스트 분류) → 상담사 큐로 라우팅.

### 축 B. 사고 후 회수 — HUG (소주제①)

**B1. 채권회수 대시보드(Recovery Dashboard)** — `/hug` 대시보드
- 채권 포트폴리오 현황: 상품·지역·주택유형·금액구간별 대위변제 잔고(합성데이터 시각화).
- 건별 카드: **예상 회수율(회귀+LOW/MED/HIGH 등급) / 예상 배당 소요기간 / 회수 우선순위 스코어** + SHAP 요인 설명("채권구분=구상채권, 물건=다세대 → 회수율 하향").
- **회수 시뮬레이터(발표 킬러 장면)**: "우선순위 스코어 순 처리 vs 접수순 처리"를 슬라이더로 비교 → 동일 처리용량에서 회수액·기간 개선폭을 애니메이션 차트로 표시. "합성데이터 기준 시뮬레이션" 라벨 부착.
- 유사사건 조회: 벡터/조건 검색으로 과거 유사 채권의 실제 회수 경로 표시.

**B2. 사고 접수→회수 타임라인**: 임차인 사고 접수(/incidents) → HUG 큐 유입 → 경공매·배당 진행 단계 추적 화면(포털의 피해지원 절차를 UI로 재현).

### 축 C. 계약 체결 — 임차인×임대인 공동

**C1. 안심 전자계약**: 양측 로그인 → 공동 세션(폴링/웹소켓) → 조건 합의 체크 → **AI 특약 추천**(위험진단 결과·상담사례에서 자주 등장한 분쟁 예방 특약 자동 제안 — 임차in '특약 대사전'의 AI 버전) → 서명 → 계약서 해시 블록체인 앵커(mock tx) → 위변조 검증 화면(해시 재계산 일치/불일치 데모).
- 스토리 연결: "계약 원본의 무결성이 사고 후 분쟁·회수 단계의 증거력이 된다" → 축 B와 연결.

**C2. 거주 중 모니터링(경량)**: 등기 변동·보증만기 알림 목업(/notifications) — 임차in 'AI 알리미'와 접점.

## 4. ML 개발 계획 (데이터 한계 준수)

| 모델 | 타겟 | 데이터 | 방법 |
|---|---|---|---|
| 예상 회수율 | 발생금액대비 총회수금액(%) | 배당내역 28,961 | LightGBM 회귀 + LOW/MED/HIGH 등급화, SHAP |
| 예상 소요기간 | 신청일자대비 배당 소요일(음수 38건 제외, 산정기준 발제사 문의 병행) | 배당내역 | LightGBM 회귀(분위 예측), SHAP |
