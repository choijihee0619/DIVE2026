# 정상대조군 데이터 확보 방안 (260721 조사)

## 1. 배경

"사고 발생확률" 모델은 의도적으로 제외되어 있다 — 발제사 제공 데이터 전체가 사고 사례라
**정상(무사고) 대조군이 없어** 이진분류 학습·검증이 불가능하기 때문. 현재는 HOUSTA
발급현황(모수) × 사고현황(분자) 결합으로 지역×기간 실제 사고율을 통계적 사전확률로 사용 중.

## 2. HOUSTA 실조사 결과 (2026-07-21, 직접 열람)

HUG 빅데이터 개방 포털 `공공데이터 > 전세 및 임대보증` 카테고리 **20개 데이터셋 전수 확인**.

| 후보 데이터셋 | 단위 | 판정 |
|---|---|---|
| 전세보증금반환보증 발급현황 | 시도×연월×주택유형 집계 | 이미 수집(6,906행). 건별 아님 |
| 전세보증금반환보증 **상세현황** (data.go.kr 15033824) | 상품×연월×주택유형×시도 집계(건수·보증금액) | "상세"라는 이름과 달리 **집계** |
| 전세반환보증금보증 **세부내역** (data.go.kr 15134038) | 연도별 대위변제·회수 총계 | **집계** (연 6행 수준) |
| 지역별 전세보증금반환보증 사고현황 | 시군구 집계 | 이미 수집(272행) |
| 나머지 16종 | 전부 집계 또는 상품 안내 | 해당 없음 |

**결론: HOUSTA(및 data.go.kr 연계분)에서 건별 정상 보증 데이터는 공개되지 않는다.**
개인정보(임차인·임대인·주소 식별 가능성) 때문에 건별 발급 데이터는 비공개가 원칙이다.

## 3. 솔루션 3안

### A안 (권장 PoC) — RTMS 전월세 실거래를 유사 대조군으로 쓰는 case-control 설계

국토부 실거래가 공개시스템의 **전월세 신고 건별 데이터**(아파트·연립다세대·단독다가구·오피스텔
4종 API)는 계약 단위로 보증금·지역(법정동)·주택유형·면적·계약연월을 제공한다. 전월세 계약의
절대다수는 정상 계약이므로 이를 **유사 정상군(pseudo-controls)** 으로 표집하고, 사고군(합성
사고현황 25,687 또는 HOUSTA 사고 분포)과 결합하면 사고확률 판별 모델을 만들 수 있다.

- 공통 피처: 지역(시군구), 주택유형, 보증금 구간, 계약 시점, **전세가율**(보증금 ÷ VWorld
  공시가격 — 260721 live 연동 완료로 계산 가능), 지역 사고율 사전확률
- 방법론 주의(발표 명시 필수):
  - 대조군 라벨 노이즈: 전월세 계약 일부는 실제로 사고가 났을 수 있음 (사고율 수 % 수준이라 영향 제한적)
  - 표본 편향: 보증 가입 모집단 ≠ 전체 전월세 모집단 → 지역×유형 층화 표집으로 완화
  - 사고군이 합성데이터면 "합성+실거래 혼합 기준, 실데이터 재학습 전제" 명시
- **필요 조치(사용자 작업)**: data.go.kr에서 기존 `DATA_GO_KR_API_KEY`로 전월세 4종
  활용신청(자동승인). 260721 검증 결과 키 자체는 유효(매매 API 200 OK)하나 전월세는
  미신청 상태라 403.
  - 아파트 전월세: `RTMSDataSvcAptRent/getRTMSDataSvcAptRent`
  - 연립다세대: `RTMSDataSvcRHRent` / 단독다가구: `RTMSDataSvcSHRent` / 오피스텔: `RTMSDataSvcOffiRent`

### B안 (현행 유지·보강) — 집계 기반 사전확률 + rule-based 충족도

지금 구조. 통계적으로 가장 정직하며 심사 방어력이 높다. A안 도입 후에도 지역 사고율
사전확률은 모델 보정(calibration) 기준으로 계속 사용한다.

### C안 (요청 트랙) — HUG 비식별 건별 데이터 제공 요청

HOUSTA `데이터 신청 및 활용 > 공공데이터 신청`(p050206.jsp) 경로로 비식별 건별 발급
데이터를 요청. 해커톤 기간 내 회신 불확실 — 발표 시 "실데이터 확보 로드맵"으로 언급.

## 4. 권장 실행 순서

1. (즉시) 발표 자료에는 B안 논리 유지 — "정상대조군 부재 → 사고확률이 아닌 고위험 패턴
   충족도" 스토리가 오히려 데이터 정직성 차별점
2. (사용자, 5분) data.go.kr 전월세 4종 활용신청 → 승인 즉시 A안 PoC 수집 가능
3. (승인 후) 층화 표집 스크립트 + LightGBM 이진분류 PoC → 기존 `train_ml_models.py`
   파이프라인에 `accident_clf` 추가, 지표에 한계 명시
4. (병행) C안 신청서 제출
