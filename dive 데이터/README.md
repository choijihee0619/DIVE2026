## 3. 발제자료 PDF

### 무엇을 보여주는가

`AI 기반 안심 주거 생태계`를 주제로, HUG의 공공 보증데이터와 아이엔의 민간 임대차 상담데이터를 결합하여 다음을 해결하는 과제를 설명한다.

1. 계약 전 위험 신호와 상담 수요를 파악한다.
2. 사고 후 대위변제·경매·배당 패턴을 활용해 채권관리 효율을 높인다.

### PDF 설명과 실제 파일의 관계

PDF의 데이터 설명은 상위 개념과 활용 아이디어에 가깝고, 실제 CSV는 해커톤용으로 일부 컬럼만 추출·합성한 형태로 보인다. 특히 다음 항목은 실제 파일에서 확인되지 않는다.

- 계약→사고→대위변제→경매→배당을 연결하는 공통 사건 ID
- 정상 종료 계약 데이터
- 임대인의 반복 사고를 연결하는 비식별 임대인 ID
- 경매 감정가, 유찰 횟수, 회차별 최저가, 낙찰가·낙찰일
- 계약 시점의 등기부, 선순위채권, 세금, 시세 스냅샷

- 지역·보증금구간·계약상태별 상담 현황 대시보드
- 상담 내용에서 핵심 권리·분쟁 키워드 추출
- 근거가 없을 때 답변하지 않는 상담 보조 시스템

### 인사이트·주의사항

- 경매·공매(212건), 보증금 미반환(183건), 전세사기(119건)가 주요 상담 수요다.

- `전출선근저당후`는 1건뿐이므로 독립 클래스 학습이 어렵다.
- 자유서술문은 자동 정규식 검사와 별개로 실명·회사명·상황 조합 등 준식별자의 사람 검수가 필요하다.
- 특정 상담자나 임대인을 추정하는 재식별 시도는 금지한다.

## 12. 파일 간 연결 관계

### 현재 가능한 것

- 각 파일 내부에서 상품·지역·주택유형·금액구간별 분포를 분석한다.
- 경매현황 파일 내에서 `배당금/대위변제금`을 계산한다.
- 배당내역 파일의 제공 회수율·소요일을 각각 모델 타겟으로 사용한다.
- 상담데이터는 별도의 RAG·텍스트 분류에 사용한다.

### 현재 하면 안 되는 것

- 주소·금액·날짜가 비슷하다는 이유로 서로 다른 파일의 행을 같은 사건으로 연결하지 않는다.
- 상세주소를 외부 정보와 결합해 특정 임대인·개인·법인을 추정하지 않는다.
- 사고 사례만으로 정상·사고를 구분하는 `전세사고 발생확률` 모델을 만들지 않는다.
- 합성데이터 학습 점수를 실제 운영 성능으로 설명하지 않는다.

## 14. 발제사에 확인할 사항

1. PDF의 데이터 세부정보·컬럼과 실제 합성 CSV 컬럼이 다르다. 해커톤 당일 추가 데이터가 제공되는지 확인한다.
2. 사고·대위변제·경매·배당 파일을 연결하는 HUG 내부 ID가 있는지, 있다면 새로 생성한 비식별 ID로 제공 가능한지 문의한다.
3. 정상 종료 보증계약을 집계·합성 형태로 제공할 수 있는지 확인한다.
4. `보증종료일자 기준 사고발생일자 소요일`의 음수 의미와 `신청일자대비 배당 소요일`의 정확한 산정 기준을 확인한다.
5. 경매현황의 배당금 0원이 미회수·진행중·미배당 중 어느 상태인지 확인한다.
6. 배당금이 대위변제금보다 큰 행과 주택가액 극단값이 합성 규칙상 의도된 것인지 확인한다.

## 15. 최종 요약

> 실제 비식별 상담데이터는 RAG·상담분류에 직접 활용할 수 있다. HUG 합성데이터는 사고·대위변제·경매·배당 사례 내부의 금액·회수율·처리기간 패턴과 시연용 모델에 활용할 수 있다. 다만 정상 계약과 공통 사건 ID가 없으므로, 실제 사고확률이나 사건 전 과정 통합 예측으로 과장해서는 안 된다.
